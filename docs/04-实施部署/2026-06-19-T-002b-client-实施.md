# T-002b client 实施说明（2026-06-19）

> 形态 B：`cpr <tool> [args …]` 的客户端落地手册。范围严格按 T-002b agent2 派发：
> `cpr/cli/render.py`、`cpr/i18n/`、`README.md`、本文件。
> 协议 / 退出码 / 配置字段以 [`docs/02-系统设计/2026-06-19-AI起手式-架构与协议.md`](../02-系统设计/2026-06-19-AI起手式-架构与协议.md) 为唯一来源。

## 1. 安装与首次跑通

```bash
python -m pip install -e ".[dev]"     # Python ≥ 3.11
cpr --diag                            # 自检：版本、locale、cache 目录、endpoint
cpr                                   # 无参时打印用法说明（exit 0）
```

`pip install -e` 会注册 `cpr` 控制台脚本（`pyproject.toml` `[project.scripts]`）。脚本等价于
`python -m cpr.cli.main`，**没有**任何 TUI / curses / prompt_toolkit 依赖。

## 2. 起本地 mock server

正式环境还没上线前，使用仓库内 mock：

```bash
python scripts/mock_server.py                # 监听 127.0.0.1:8765（默认）
PORT=18765 python scripts/mock_server.py     # 端口冲突时换端口
```

mock server 由 `tests/fixtures/server/scenarios/*/response.json` 与 `tests/fixtures/server/errors/*` 驱动，
`scripts/mock_server.py` 只用 stdlib `http.server`，启动即就绪、无依赖。

`bash scripts/manual_sdk_smoke.sh` 会自动起 mock、跑 5 条代表性命令、再清理 mock 进程，是手工冒烟的入口。

## 3. 切换到自部署 / 官方 server

写入 `~/.cpr/config`（YAML）：

```yaml
server:
  endpoint: https://your-cpr-server.example.com
  timeout_seconds: 5
client:
  help_timeout_seconds: 2
  locale: auto                # auto | zh-CN | en-US
  confirm_danger: always      # always | once | never
cache:
  dir: ~/.cpr/cache
```

环境变量优先：

| 变量 | 作用 |
| ---- | ---- |
| `CPR_HOME`  | 整体重写 `~/.cpr`（用于沙箱、CI、容器） |
| `CPR_LOCALE`| 一次性强制 locale（`zh-CN` / `en-US`） |
| `NO_COLOR`  | 任意非空值即关闭 ANSI |
| `LANG`      | locale 兜底（zh* → zh-CN，否则 en-US） |

## 4. 渲染契约（client 这一侧）

`cpr/cli/render.py` 负责形态 B §3 的 [1]–[5] 输出和 [6] 危险横幅，全部经由 `cpr/i18n/` 取串：

- 颜色：纯 ANSI（`\x1b[...m`），无 rich/click/textual。`NO_COLOR=1`、stdout 非 TTY、调用方传 `color=False`
  任一成立时整段 plain。
- 表格：3 列（TOKEN / TYPE / DESCRIPTION），双空格列间距 + 一条 `dim` 连字符规则线，**无 Unicode 框线**。
  CJK / 全角字符按 2 列宽对齐。超过 `max_candidates`（默认 10）折叠为 `… and N more (folded)`。
- 危险确认：`confirm_danger()` 默认答案 N；输入 `y` / `Y` 才返回 True，`yes`、空串、Ctrl-D（EOFError）都为 False。
  `--yes` / `-y` / `client.confirm_danger=never` 时 `yes=True` 直接跳过 prompt。
- quota 文案：`render_quota_status()` 只读 `quota.reset_in_seconds`，按 `(s+3599)//3600` 向上取整为整小时数，
  避免本地时区/时钟依赖（协议 P1-2）。
- 错误渲染：`render_error_payload()` 覆盖协议 §2.1 全部 11 个错误码（title + hint）；未识别 code 走
  `error.generic_title` 兜底，附带 `message`。

## 5. i18n 双语对齐

`cpr/i18n/zh-CN.yaml` 与 `cpr/i18n/en-US.yaml` 是 dotted-key flat YAML，**任何新 key 必须双语同时落**。
`I18n.t(key, **fmt)` 顺序：当前 locale → en-US 兜底 → 返回原 key（同时 WARNING 日志）。
locale 解析顺序：`CPR_LOCALE` > `~/.cpr/config client.locale`（`auto` 视作未设） > `LANG` > `en-US`；
凡非 zh* 一律回落 en-US（MVP 只发两套）。

测试在 `tests/cli/test_render.py::test_locales_have_each_error_code` 用参数化保证 11 个 code 的 title/hint
两套都齐。

## 7. 排错

| 现象 | 排查路径 |
| ---- | -------- |
| `NETWORK_UNREACHABLE` | `~/.cpr/config server.endpoint` 是否正确；mock server 是否还在跑 |
| `HELP_TIMEOUT`        | 调高 `client.help_timeout_seconds`；或检查目标工具 `--help` 是否卡住 |
| 中文乱码              | 终端 UTF-8 / `LANG` / `LC_ALL`；`CPR_LOCALE=zh-CN` 强制 |
| 表格列对不齐          | 终端字体非等宽 CJK（`_visible_width()` 已按 East Asian Width 表估算，仅做 MVP 近似） |
| 危险确认默认变成 y    | 检查是否被 `--yes` / `confirm_danger=never` 短路 |

`cpr --diag` 会回显 client_id / locale / endpoint / 上次 `prompt_version`，是上报问题时最有用的一行。
