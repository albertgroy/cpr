# 循环验证方案

目标：用短循环验证“命令工作台”体感，而不是一次性大改。

## 循环节奏

每轮循环固定包含：

1. 定义一个可体验目标。
2. 实现最小功能。
3. 用户实际操作。
4. 收集体感反馈。
5. 修正模型或交互。
6. 单独提交。

## Loop 1：静态命令构造

目标：验证 token 追加/删除和层级说明是否自然。

范围：

```text
sdk
sdk list
sdk list java
sdk install
sdk install java
```

不执行真实命令，只展示说明和候选。

验证问题：

- 用户是否理解“点击就是追加 token”？
- 返回是否自然等同于删除上一段？
- 当前命令是否始终清晰？
- 内容区说明是否足够？
- 是否比菜单体感更轻？

通过标准：

- 用户能不看外部文档构造出 `sdk list java`。
- 用户能自然返回到 `sdk list` 或 `sdk`。

## Loop 2：执行只读命令

目标：验证执行结果展示。

范围：

```text
sdk list
sdk list java
sdk current java
```

验证问题：

- 执行结果和说明是否混乱？
- 结果是否容易选择复制？
- 上一次结果是否能保留给 `/fix` 或 `/ai`？
- 大输出是否需要折叠或分页？

通过标准：

- 用户能执行 `sdk list java` 并看到完整输出。
- 用户能复制结果中的 Identifier。

## Loop 3：动态候选

目标：从真实命令输出中提取下一步候选。

范围：

- 解析 `sdk list java` 的 Identifier。
- 在 `sdk install java` 下一层展示 Identifier 候选。

验证问题：

- 候选数量很多时如何显示？
- 是否需要搜索过滤？
- 是否需要推荐 LTS？
- 是否需要按发行版过滤，例如 tem/ms/amzn？

通过标准：

- 用户能从候选中选出 `17.0.19-tem`。
- 命令行变成 `sdk install java 17.0.19-tem`。

## Loop 4：执行会修改环境的命令

目标：验证风险提示和执行确认。

范围：

```text
sdk install java <identifier>
sdk default java <installed-version>
```

验证问题：

- 是否需要危险等级？
- 确认提示是否足够轻？
- 安装进度如何显示？
- 失败后是否能保留错误上下文？

通过标准：

- 用户知道命令会下载并写入 `~/.sdkman`。
- 用户能看到安装过程和失败原因。

## Loop 5：Slash command AI

目标：验证 AI 作为主输入区特殊命令，而不是独立聊天窗口。

范围：

```text
/ai <问题>
/explain
/fix
/help
```

验证问题：

- 用户是否能自然想到用 `/ai` 提问？
- `/explain` 是否能解释当前命令？
- `/fix` 是否能分析上一条错误？
- AI 建议命令如何被采用？

通过标准：

- 用户能用 `/ai 帮我安装 Java 17 LTS` 得到可执行命令。
- 用户能用 `/fix` 理解 SDKMAN 报错。

## Loop 6：多语言切换

目标：验证 i18n 不是后补。

范围：

- `zh-CN`
- `en-US`

验证问题：

- 命令本身是否保持不翻译？
- 标题、说明、帮助是否完整切换？
- 缺失翻译 fallback 是否合理？

通过标准：

- 同一个命令节点能显示中文和英文说明。

## 每轮通用检查清单

实现检查：

- 是否保持真实命令可见？
- 是否避免隐藏副作用？
- 是否支持键盘操作？
- 是否支持鼠标点击候选？
- 是否能复制关键内容？
- 是否有测试？
- 是否单独提交？

用户体感检查：

- 是否比原 CUI 菜单轻？
- 是否比直接看 help 更容易？
- 是否没有把 CLI 变成另一套复杂 UI？
- 是否能兼顾新手和熟手？

## 验证环境

优先验证：

1. macOS ARM64。
2. Ubuntu ARM64。
3. Linux x86_64。

SDKMAN 特别需要验证：

- `sdk` 是 shell function，不是普通二进制。
- 必须 source `~/.sdkman/bin/sdkman-init.sh`。
- ARM64 平台的 Java Identifier 列表可能与 x86_64 不同。

## 提交策略

每完成一个独立问题立即提交：

- `feat(core): add command node model`
- `feat(tui): add prompt workspace layout`
- `feat(sdkman): add sdk list command tree`
- `fix(sdkman): parse java identifier column`
- `feat(ai): add slash explain command`

不要把多个独立问题堆在一个 commit。
