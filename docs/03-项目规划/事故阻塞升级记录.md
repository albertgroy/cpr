## 升级触发条件

- 重复（≥2 次）测试失败且原因不明。
- CCB 通信异常：ask 无响应 / 回调丢失 / 队列阻塞。
- Provider 故障：claude / opencode / codex / kiro-cli 中任一不可用。
- 发布失败或回滚。
- 需求与契约冲突无法协调。

## 处理路径

- 先记录现象与影响，避免直接重启或清空上下文。
- 诊断 CCB 状态（参考 `ccb-self-diagnose` 技能）。
- 决策选项：暂停 / 重试 / 重派 / 恢复 CCB 上下文 / 咨询用户。
- 不要随手使用清空记忆的重置路径（如 `ccb -n`），除非用户明确选择。
