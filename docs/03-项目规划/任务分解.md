> 本文件用于把"阶段任务"进一步分解为可派发到开发对（agent1+agent2 / agent3+agent4）的 CCB 任务卡片。

## 当前活跃阶段

**形态调整（2026-06-19）**：T-001 实现的 prompt_toolkit 工作台 + 本地 yaml 命令树形态被用户首次实测后否决。
新形态：cpr 作为 AI 起手式 CLI（一次性调用、回车后显、server 端 LLM 解析 help_text）。
新阶段：T-002a 协议契约 → T-002b client（含 mock server）→ T-003 server（含本地启动脚本）。
依据：`docs/01-需求分析/2026-06-19-形态-B-需求.md` 第二版、`docs/02-系统设计/2026-06-19-AI起手式-架构与协议.md`。

## 任务卡片格式

每个任务卡片包含：

- 任务 ID（T-NNN）
- 标题
- 关联阶段
- 目标
- 范围 / 文件
- 假设
- 期望产出
- 验证方式
- 负责开发对
- 依赖任务
