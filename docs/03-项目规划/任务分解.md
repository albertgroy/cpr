## 任务卡片

### T-001 SDKMAN MVP 命令构造器

- 阶段：阶段 1 + 阶段 2 + 阶段 3
- 目标：实现 `sdk` / `sdk list` / `sdk list java` / `sdk install java <id>` / `sdk use|default|current java <ver>` 的 token 构造、动态候选与执行。
- 关联评审：`docs/02-系统设计/2026-06-19-契约评审-archi.md`（archi job_bee22322e9d9）。
- 工作目录：`/Users/leel/my_work/cpr`。
- 技术栈：Python 3.11+、prompt_toolkit、pyyaml、pytest、pytest-asyncio。其余依赖按 `docs/03-项目规划/环境密钥与依赖变更规约.md` 走变更流程。
- 分支：`feat/T-001-sdkman-mvp`，单分支提交，按下方提交顺序拆分 commit。

#### 范围（按 archi 评审约束）

- 核心模型：CommandNode、CommandSession、CandidateProvider（静态 + 动态）、Executor（direct + bash-lc-source）、SlashCommand parser、i18n loader。
- 数据：`data/sdkman.yaml` 命令树。
- TUI：`prompt_toolkit` 输入区 + 内容区 + 候选区，键盘绑定 Enter/Backspace/Esc/Tab/方向键。
- 测试：`tests/core/`、`tests/data/`，覆盖率门槛 `cpr.core ≥85%`。
- 脚本：`scripts/manual_sdk_smoke.sh`。
- 诊断：`python -m cpr --check-tree` / `--slash-parse <text>`。

#### 文件边界

- agent1：`cpr/core/`、`tests/core/`、`scripts/`、`pyproject.toml`（核心依赖与 pytest 配置）
- agent2：`cpr/tui/`、`data/sdkman.yaml`、`tests/data/`、`README.md`
- 共写：`cpr/i18n/`（loader + zh-CN/en-US 文案表）

#### 提交顺序（每条独立 commit）

1. `feat(core): add command node and session models`
2. `feat(core): add static candidate provider`
3. `feat(core): add executor with bash-lc-source mode`
4. `feat(data): add sdkman command tree yaml`
5. `feat(tui): prompt workspace layout (input + content + candidates)`
6. `feat(core): add dynamic candidate provider for sdk list java identifiers`
7. `feat(core): add slash command parser with /back /clear /help`
8. `feat(tui): wire enter/backspace/esc/tab/arrow keybindings`
9. `chore(docs): contract patch v1 covering identified gaps`
10. `chore(scripts): manual sdk smoke script`

#### 期望产出

- 可运行原型：`python -m cpr` 进入工作台，构造 7 条 SDKMAN 命令，token 追加 / 回退 / 执行 / 显示结果流畅。
- 动态候选支持 Identifier 与已安装版本，并在执行修改环境的命令后失效缓存。
- 契约补丁 v1 写入 `docs/02-系统设计/`，覆盖 archi 6 项缺口。
- 验收报告：`docs/05-验收反馈/2026-06-XX-T-001-验收报告.md`，含 macOS ARM64 + Ubuntu ARM64 手工记录。

#### 验证命令

```bash
python -m pip install -e ".[dev]"
pytest -q
pytest --cov=cpr.core --cov-fail-under=85
python -m cpr --check-tree
python -m cpr --slash-parse "/ai 帮我安装 Java 17 LTS"
bash scripts/manual_sdk_smoke.sh
```

#### 不通过条件（覆盖契约 + archi 追加项）

- 任一核心场景验收点失败。
- Executor 静默吞错或丢失 stderr。
- Executor 在网络不可达时返回 `ok=true`。
- prompt_toolkit 主循环在 `sdk list java` 期间 ≥1s 卡顿（异步契约违反）。
- i18n fallback 缺失。
- 引入了禁止的依赖或真实 LLM 调用。

#### 禁止

- 不引入真实 LLM 调用（`/ai` `/explain` `/fix` 走 mock provider，留 T-003）。
- 不引入 textual / click / rich / pty。
- 不在仓库内提交 `.env` 或 API key；缺失 secret 向用户索取。
- 两位 agent 不并发编辑同一文件；遵守上文文件边界。

#### 依赖

- 无前置任务。
- 评审前置已完成（archi 选项 B：不阻塞）。

#### 负责开发对

- agent1（claude） + agent2（opencode）。
- 一对内角色：agent1 主实现核心 + 测试 + 脚本；agent2 主实现 TUI + 数据 + 用户文档；互为同伴评审。

#### 回调要求

- 阶段：阶段 5（提前打地基，避免后期重构）
- 目标：定义命令节点 / UI 固定文案 / slash command help 的 i18n 数据结构，缺失语言 fallback 到 `en-US` 或 `zh-CN`；locale 解析顺序 `CPR_LOCALE` > 用户配置 > `LANG` > `en-US`。

- 期望产出：能在启动时切换 `zh-CN` / `en-US`，命令树展示对应文案。
- 验证：单元测试覆盖 fallback 路径与 caplog 警告。
- 负责开发对：T-001 完成首版后，agent3 + agent4 接入。
- 依赖：T-001 的 CommandNode 结构与 i18n loader 已稳定。
