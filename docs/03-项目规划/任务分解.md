# 任务分解

> 高层任务结构沿用 `docs/03-项目规划/任务规划.md` 中阶段 0–7 的拆分。
> 本文件用于把"阶段任务"进一步分解为可派发到开发对（agent1+agent2 / agent3+agent4）的 CCB 任务卡片。

## 当前活跃阶段

阶段 0 → 阶段 3：项目命名 + 核心模型 + TUI 原型 + SDKMAN 试点。

## 任务卡片格式

每个任务卡片包含：

- 任务 ID（T-NNN）
- 标题
- 关联阶段
- 目标
- 范围 / 文件
- 假设
- 期望产出
- 验证方式
- 负责开发对
- 依赖任务

## 任务卡片

### T-001 SDKMAN MVP 命令构造器

- 阶段：阶段 1 + 阶段 2 + 阶段 3
- 目标：实现 `sdk` / `sdk list` / `sdk list java` / `sdk install java <id>` / `sdk use|default|current java <ver>` 的 token 构造与执行。
- 范围：核心模型（CommandNode、Session、CandidateProvider、Executor）+ TUI 原型 + SDKMAN 命令树。
- 假设：暂选 prompt_toolkit；`bash -lc 'source ~/.sdkman/bin/sdkman-init.sh && sdk ...'` 执行 shell function。
- 期望产出：可运行原型；进入 / 回退 / 执行 / 显示结果 流畅；动态候选支持 Identifier 与已安装版本。
- 验证：单元测试 + 录屏 / 文本交互记录 + 在 macOS 与 Linux 上手工跑一次 `sdk list java`。
- 负责开发对：agent1 + agent2（小步快走）。
- 依赖：无。

### T-002 i18n 文案机制

- 阶段：阶段 5（提前打地基，避免后期重构）
- 目标：定义命令节点 / UI 固定文案 / slash command help 的 i18n 数据结构，缺失语言 fallback 到 `en-US` 或 `zh-CN`。
- 范围：locale 加载、节点 schema、UI 固定文案表。
- 期望产出：能在启动时切换 `zh-CN` / `en-US`，命令树展示对应文案。
- 验证：单元测试覆盖 fallback 路径。
- 负责开发对：T-001 完成首版后，agent3 + agent4 接入。
- 依赖：T-001 的 CommandNode 结构已稳定。
