# 环境密钥与依赖变更规约

## 环境分隔

原型阶段先只区分两个环境：

- `local`：开发者机器（macOS / Linux），用于 prototype 调试与手工验证 SDKMAN 命令。
- `ci`：未来引入 GitHub Actions 等 CI 时启用，用于跑单元测试与跨平台 smoke。

`staging` / `production` 在产品正式发布前不启用，避免提前承诺不稳定接口。

## 密钥与 Secret

- 真实 `.env`、API key、token 不进入 git、文档、日志、prompt。
- 必需变量名通过 `.env.example` 维护（首次需要时再创建）。
- AI slash command（`/ai` / `/explain` / `/fix`）涉及外部 LLM API key 时：
  - 通过环境变量读取，不写入仓库。
  - 文档只提及变量名，不提及具体值。
- 用户提供 secret 给某个 agent 时，仅在该次对话中使用，不持久化。

## 依赖变更

任何新增 / 升级 / 删除依赖前，提交说明：

- 原因（解决的问题、替代方案对比）
- 影响环境（local / ci）
- lockfile 评审责任人
- 兼容性风险（Python 版本 / OS / shell function 调用）
- 验证命令（构建 + 单元测试 + SDKMAN 手工验证）
- 回滚方案

原型阶段尤其注意：

- TUI 库（prompt_toolkit / Textual / blessed）选型确定前，不引入任何交互框架依赖。
- 不为单一便利函数引入大型库。
