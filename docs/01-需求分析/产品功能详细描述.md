# 产品功能详细描述

## 背景

传统 CLI 工具有两个明显问题：

1. 命令层级深、参数多，用户需要频繁查 help、文档或搜索。
2. 图形化封装容易把简单命令变成另一套重交互，用户失去对真实命令的掌控感。

本项目要解决的不是“重新实现工具”，而是“帮助用户理解、构造、执行真实命令”。

## 产品定位

一个交互式命令工作台。

用户面对的是一个类似 Claude / 终端的界面：

- 上方内容显示区：展示当前命令层级说明、候选项、执行结果、AI 解释。
- 下方主输入区：用户输入真实命令或 slash 命令。

普通输入代表命令构造或执行；slash 命令代表辅助能力。

## 核心体验

### 1. 命令作为主输入

示例：

```text
> sdk
```

界面展示：

```text
当前命令：sdk

说明：
sdk 是 SDKMAN 的入口命令，用于管理 JVM 生态工具版本。

可选下一段：
  list      列出 candidate 或某个 candidate 的版本
  install   安装指定 candidate/version
  use       当前 shell 临时切换版本
  default   设置默认版本
  current   查看当前版本
  home      输出安装路径
```

用户可以继续输入：

```text
> sdk list
```

也可以点击或用方向键选择 `list`，效果等同于把 `list` 追加到输入区。

### 2. 每一层都显示功能说明

用户不需要每层手动输入 `help`。

当命令变为：

```text
> sdk list
```

界面展示：

```text
当前命令：sdk list

说明：
list 用于列出 SDKMAN 支持的 candidate；继续指定 candidate 后，可列出该 candidate 的可安装版本。

可选下一段：
  java      JDK 版本
  maven     Maven 构建工具
  gradle    Gradle 构建工具
  kotlin    Kotlin 编译器
  scala     Scala 语言
```

当命令变为：

```text
> sdk list java
```

界面展示：

```text
当前命令：sdk list java

说明：
列出当前平台可用的 Java 版本 Identifier。安装时应使用 Identifier，例如 17.0.19-tem。

可执行：
  Enter 执行当前命令
  Backspace 删除上一段
```

### 3. 返回就是删除上一段 token

当前命令：

```text
> sdk list java
```

返回后变成：

```text
> sdk list
```

再返回：

```text
> sdk
```

这比传统菜单“返回上级”更符合命令行用户心智。

### 4. 执行结果在同一内容区展示

执行命令后，内容区展示：

```text
命令：sdk list java

结果：
Available Java Versions for Linux ARM 64bit
...
```

不再使用独立 log 蓝框作为主要交互。用户可自然选择内容区文字；必要时可以提供纯文本结果视图。

### 5. AI 是 slash command，不是独立聊天窗口

AI 不再作为独立对话区存在，而是主输入区里的特殊命令。

建议第一批 slash 命令：

```text
/ai <问题>       带当前命令上下文向 AI 提问
/explain        解释当前命令
/fix            分析上一条失败命令和错误输出
/help           显示当前层级能力
/clear          清空内容区
/back           删除上一段 token
```

示例：

```text
> sdk install java 17.0.19-tem
> /explain
```

AI 输出：

```text
该命令通过 SDKMAN 安装 Java 17 的 Temurin 发行版。它会下载文件并写入 ~/.sdkman/candidates/java。
```

示例：

```text
> /ai 帮我安装 Java 17 LTS
```

AI 可返回建议命令：

```text
推荐命令：sdk install java 17.0.19-tem
原因：这是 SDKMAN 当前平台可用的 Temurin Java 17 LTS Identifier。
```

用户确认后可将建议命令填入主输入区。

## 关键设计原则

1. 真实命令优先：界面展示和执行的都是真实 CLI 命令。
2. 不重造工具：不重新实现 sdk/git/docker/pyenv，只提供命令知识、提示和执行辅助。
3. 层级即 token：每次选择就是追加一个命令 token。
4. 返回即删除：回退就是删除最后一个 token。
5. 说明常驻：当前层级的功能说明始终在内容区可见。
6. AI 上下文化：AI 围绕当前命令、上次结果、上次错误工作。
7. 多语言内建：菜单标题、说明、帮助从一开始就支持 i18n。

## 初始试点范围

第一版只做 SDKMAN 试点，不迁移所有旧功能。

支持命令树：

```text
sdk
  list
    java
    maven
    gradle
    kotlin
  install
    java
      <Identifier from sdk list java>
  use
    java
      <installed version>
  default
    java
      <installed version>
  current
    java
```

## 非目标

第一阶段不做：

- 完整替代 shell。
- 完整支持所有 CLI 工具。
- 复杂 IDE 式图形界面。
- 把所有操作包装成自定义业务模块。

第一阶段重点是验证命令构造器体感。
