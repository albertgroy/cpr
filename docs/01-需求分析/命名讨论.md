# 命名讨论

当前目录名暂定：`cpr`。

这个名字可以作为工作代号，但是否适合作为正式产品名需要讨论。

## CPR 的可能含义

### Command Prompt Runner

优点：

- 和命令行相关。
- 表示运行命令。

缺点：

- 过于强调 runner，但项目重点不是执行，而是提示、构造、理解命令。

### Command Path Recommender

优点：

- 突出“命令路径推荐”。
- 符合逐层构造 token 的产品形态。

缺点：

- Recommender 偏推荐系统，可能弱化执行工作台属性。

### Command Pilot Runtime

优点：

- 有“驾驶/辅助操作命令”的感觉。
- Pilot 表示用户仍然掌控方向。

缺点：

- Runtime 含义偏技术底层，不够直观。

### Command Prompt Router

优点：

- 表示从当前 prompt 路由到下一层命令。

缺点：

- Router 不够产品化。

## CPR 的问题

1. `cpr` 常见含义是心肺复苏，搜索和认知会混淆。
2. 不直接表达 command builder / command assistant。
3. 三字母缩写不利于早期理解。
4. 如果作为开源项目名，可能已有大量重名项目。

## 候选命名

### cmdpilot

含义：Command Pilot。

优点：

- 用户是驾驶员，工具是副驾驶。
- 和 AI 辅助也兼容。
- 容易理解。

缺点：

- 可能已有重名。

### cmdguide

含义：Command Guide。

优点：

- 强调引导用户掌握命令。
- 清晰直接。

缺点：

- 略普通。

### cmdpath

含义：Command Path。

优点：

- 对应逐层 token 路径。
- 短。

缺点：

- 不强调执行和 AI。

### promptpath

含义：Prompt + Path。

优点：

- 和主输入区/命令路径都相关。
- 有探索路径的感觉。

缺点：

- 可能让人以为是 AI prompt 管理工具。

### cmdworkbench

含义：Command Workbench。

优点：

- 准确表达“命令工作台”。
- 包含构造、解释、执行、AI。

缺点：

- 名字较长。

### shellpilot

含义：Shell Pilot。

优点：

- 面向 shell 命令辅助。
- 体感清晰。

缺点：

- 可能让人误解为替代 shell。

### commanddeck

含义：Command Deck。

优点：

- 有控制台/操作面板意味。
- 比较产品化。

缺点：

- 与卡片 deck 有轻微歧义。

## 当前建议

工程目录可以暂时继续用：

```text
~/my_work/cpr
```

正式名称建议优先讨论：

1. `cmdpilot`
2. `cmdguide`
3. `cmdpath`
4. `cmdworkbench`

如果强调“AI + 命令副驾驶”，推荐 `cmdpilot`。

如果强调“学习和提示命令”，推荐 `cmdguide`。

如果强调“逐层构造命令路径”，推荐 `cmdpath`。

## 命名决策标准

一个好名字应该满足：

- 能表达“命令行辅助/构造”。
- 不让人误解为替代 shell。
- 不让人误解为只做 AI chat。
- 不让人误解为只做命令执行器。
- 简短，适合 CLI 二进制名。
- 输入方便。

## CLI 名称建议

即使项目名较长，CLI 命令也应短。

可选：

```text
cpr
cmdp
cpath
pilot
cg
```

但 `pilot` 太泛，`cg` 太不清晰。若正式名为 `cmdpilot`，CLI 可为：

```text
cmdpilot
```

或短别名：

```text
cpilot
```
