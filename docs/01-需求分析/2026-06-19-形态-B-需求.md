# 形态 B 需求 — cpr 作为 AI 起手式 CLI（2026-06-19，第二版）

> 本文件覆盖第一版（同名文件）。第一版假设"本地手写 YAML 命令树"，
> 用户实测后判断该路径无法覆盖市面工具的量与版本漂移，决定改走 AI 起手式。
> 第一版形态规约（输出结构、危险确认、`cpr` 无参输出等）部分仍然保留。

## 0. 形态变化总览

| 维度 | 第一版（已废） | 第二版（现行） |
| --- | --- | --- |
| 命令解析 | 本地手写 `data/<tool>.yaml` 命令树 | client 跑 `<tool …> --help` → server LLM 解析 → 缓存 |
| 工具覆盖面 | 仅显式 yaml 化的工具 | 任意可执行 CLI（凡是有 `--help` 输出的） |
| AI | 留 T-003+，T-002 不接 | **AI 即起手式本身**，没有 AI 就没有 cpr 第一版 |
| 离线 / server 故障 | 不涉及 | 硬失败 + 自配置 server 兜底 |
| `data/sdkman.yaml` 等 | 核心数据资产 | **弃用**（server 解析全替代） |

## 1. 调用形态与基本约定（沿用第一版）

| 维度 | 结论 |
| --- | --- |
| 调用模式 | 一次性 CLI；调用后输出，进程退出 |
| 触发显示 | 用户回车后再显（不做 keystroke 联想） |
| 滚动 / 分页 | 不做 |
| 推荐参数显示 | 极简表格 + 折叠 |
| 颜色装饰 | ANSI 颜色 + 粗体 + ASCII；无 emoji，无 Unicode 框线 |
| 命令层级输入方式 | 完整命令式：`cpr sdk install java` 一次写完 |
| 危险命令确认 | 由 server 在响应中标记 `danger: true`；client 触发 `y/N` 二次确认（默认 N） |
| 未知命令处理 | 取决于 `<tool> --help` 是否成功（见 §5） |
| `cpr` 无参 | 输出总览（项目自介、AI 起手式说明、`cpr <cmd>` / `cpr --help`） |
| 文案语言 | 跟系统 locale；client 内置 zh-CN / en-US fallback |

## 2. 用户认知模型

> 我不会用大多数命令时需要去 ai 查，差了也不明白也记不住。但是有了起手式，那么就可以只记起手的名 cpr，就 3 个字符。然后所有其他命令参数权交个 cpr 来交代。

cpr 的角色：**一个把陌生 CLI 工具"现学现用"的桥**。
用户记 `cpr` 三个字，剩下让 server 端的 AI 知识库告诉他怎么写。

## 3. AI 起手式工作流

```
cpr <tool> [args …]
  ├ client 跑 <tool [子命令]> --help   （超时 2s）
  ├ key = sha256(tool + 子命令路径 + help_text)
  ├ 查 client 本地缓存 → 命中：渲染 [1]–[5]，结束
  └ miss → POST /resolve {tool, args(脱敏), help_text, locale}  （超时 5s）
            ├ server 查全局缓存 → 命中：返回 JSON（**不计配额**）
            └ miss → 调 LLM 解析 help_text + 用户意图 → 写缓存 → 返回 JSON（**计配额**）
client 阻塞 + spinner，5s 超时硬失败
若响应 danger=true → 输出 [1]–[5] + "确认执行？(y/N)" → y 才走 executor
否则结束（不自动执行）
```

## 4. 输出结构（沿用第一版 §3）

按以下顺序，缺位的段落直接省略：

```
[1] 命令头：cpr <你刚输入的全文>
[2] 一句话说明：该命令在做什么
[3] 用法行：实际可执行的真实命令样例（粗体高亮）
[4] 候选 / 子层（极简表格）：next-token / 简短说明
[5] 备注：非空才显示
[6] 危险确认行：danger=true 时追加 "确认执行？(y/N)"
```

不打印分隔线 / banner；颜色仅区分用法行 / 候选名 / 备注。

## 5. 错误与降级

| 场景 | client 行为 | 进程退出码 |
| --- | --- | --- |
| `<tool> --help` 不存在 / 失败 | 提示 "未识别的命令 / 工具未安装" | 2 |
| `<tool> --help` 超时（2s） | 同上 | 2 |
| server 不可达 | "AI 服务不可达，请稍后重试 / 检查 `~/.cpr/config` 中的 server 地址" | 3 |
| server 5s 超时 | "AI 解析超时，请重试或换更明确的命令" | 3 |
| 配额耗尽 | "今日 AI 额度已用完（X/Y），N 小时后重置；可在 `~/.cpr/config` 配置自部署 server 地址" | 4 |
| LLM 解析失败 | "AI 解析失败：<原因>" | 5 |
| 危险确认 N | 仅展示 [1]–[5]，不执行 | 0 |
| 危险确认 y / 非危险 / 用户复制粘贴 | （client 默认不替用户跑非危险命令；仅 install/use/default 类带 danger 的走 executor） | 命令本身的退出码 |

**离线兜底**：当前版本不做。本地缓存命中能离线工作，未命中即硬失败。

## 6. 隐私与脱敏

发送给 server 的 payload：`{tool, args, help_text, locale, client_version}`。

**脱敏规则**（client 出站前应用）：

- `--password`、`--token`、`--secret`、`--key`、`-p` (mysql 类)、`-i` (ssh)、`--api-key` 等 flag **后一个 token** → 替换为 `<REDACTED>`
- 形如 `user@host`、`http(s)://…`、`/Users/<name>/…`、`/home/<name>/…` 中的用户名 → 替换为 `<USER>`
- 形如 `~/.ssh/…`、`~/.aws/…`、`~/.gnupg/…` 路径 → 替换为 `<PRIVATE_PATH>`
- 形似 base64 / hex 长串（≥32 字符）→ 整段替换为 `<BLOB>`

`help_text` **不脱敏**——它是公开文档；但要文档化"我们会发送 help_text 全文"。

脱敏白名单与黑名单进 `~/.cpr/config`，用户可加私有规则。

## 7. 配额（quota）

- 默认 server 提供 50 次/日/用户的免费额度，**仅 cache miss 计数**。
- 用完即硬停（错误码 4），UX 文案明确提示重置时间 + 自部署兜底入口。
- 自部署 server 由用户在 `~/.cpr/config` 配置 `server.endpoint` 切换；
  自部署侧不强制配额（由部署方决定）。
- 用户层身份：MVP 用 `client_id`（首次启动写入 `~/.cpr/config`，UUID v4）；不接登录。

## 8. 自配置（兜底）

`~/.cpr/config` (yaml)：

```yaml
server:
  endpoint: https://cpr.<official-host>/resolve   # 默认官方
  timeout_seconds: 5
client:
  id: <uuid v4>           # 首次启动生成
  locale: auto            # auto | zh-CN | en-US
cache:
  dir: ~/.cpr/cache       # sqlite or json，T-002b 内决定
  max_entries: 1000
redact:
  extra_flags: []         # 额外脱敏 flag
  keep_user: false        # 是否保留 /home/<name>/ 中的 <name>
```

`server.endpoint` 改成 localhost 即接自部署 server（T-003 提供 docker-compose）。
