> 本文件是 T-002a 评审主体（archi 评审对象），同时是 T-002b client 与 T-003 server 实现契约。

## 1. 系统拓扑

```
┌─ user terminal ──────────────────────────────────────────┐
│  $ cpr <tool> [args …]                                   │
│                                                          │
│  ┌─ cpr client (python) ───────────────────────────────┐ │
│  │  1. 解析 argv                                        │ │
│  │  2. spawn `<tool [子]> --help` (timeout 2s)          │ │
│  │  3. redact(args)                                     │ │
│  │  4. cache.get(hash(tool, sub_path, help_text))       │ │
│  │     hit  → render                                    │ │
│  │     miss → POST /resolve (timeout 5s)                │ │
│  │  5. render [1]–[5] (+ danger y/N)                    │ │
│  │  6. 若 danger y → executor 跑真命令                  │ │
│  └──────────────────────────────────────────────────────┘ │
│              │                                            │
└──────────────┼────────────────────────────────────────────┘
               │ HTTPS (官方) / HTTP (本地)
┌──────────────▼────────────────────────────────────────────┐
│  cpr server (FastAPI 推荐)                                │
│   POST /resolve                                           │
│     ├ 校验 client_id / 配额                                │
│     ├ cache.get → hit: 返回 (不计配额)                    │
│     └ miss: LLM 解析 → cache.put → 返回 (计配额)          │
│   GET  /quota   返回当前用户用量                          │
│   GET  /health  存活探针                                  │
└───────────────────────────────────────────────────────────┘
```

## 2. 协议（HTTP/JSON）

### 2.1 `POST /resolve`

请求：

```json
{
  "schema_version": "1",
  "client_id": "uuid-v4",
  "client_version": "0.2.0",
  "locale": "zh-CN",
  "tool": "sdk",
  "args": ["install", "java", "<REDACTED>"],
  "sub_path": ["install", "java"],
  "help_text": "<原始 --help 全文，UTF-8>",
  "help_source": "sdk install --help"
}
```

字段约束：

- `schema_version`：当前固定 `"1"`，server 与 client 协议演进锚；client 与 server 不匹配 → `SCHEMA_MISMATCH` 426
- `client_id`：UUID v4；缺失或非法 → `INVALID_CLIENT` 422
- `tool`：argv[1]；不允许 `..`、`/`、空格
- `args`：argv[2:]，**已脱敏**；server 不再脱敏入参，但出站时镜像脱敏（见 §4）；记录 args 长度做异常监控
- `sub_path`：client 解析出的子命令链（参与 cache key），可空。**选择规则**：client 从最长前缀向最短前缀逐层尝试 `<tool> [sub_path...] --help`，第一条 0 退出且 stdout 非空的为准；其结果 path 即 `sub_path`。失败到根仍失败 → `HELP_NOT_FOUND`（client 本地错误，不发 server）。
- `help_text`：上限 64 KiB；**client 出站前先按 64 KiB 截断**，并按 §3.1 行尾归一规则处理；client 标 `help_truncated_at_size`（截断长度）；server 镜像回 `help_truncated`/`help_truncated_at_size` 不再二次截。
- `help_source`：实际跑的命令字符串，便于 server 日志定位

响应（2xx）：

```json
{
  "schema_version": "1",
  "cache": "hit" | "miss",
  "prompt_version": "p-2026-06-19.1",
  "quota": {"used": 12, "limit": 50, "reset_at": "2026-06-20T00:00:00Z", "reset_in_seconds": 28800},
  "result": {
    "summary": "用 SDKMAN 安装 Java",
    "usage": "sdk install java <identifier>",
    "candidates": [
      {"token": "17.0.10-tem", "desc": "Temurin 17 LTS", "kind": "id"},
      {"token": "21.0.2-tem",  "desc": "Temurin 21 LTS", "kind": "id"}
    ],
    "notes": ["首次安装会下载 JDK，约 200 MB"],
    "danger": true,
    "danger_reason": "会写入 ~/.sdkman/ 并改环境",
    "exec_template": "sdk install java {identifier}",
    "exec_template_args": {"identifier": {"from": "candidates", "kind": "id"}},
    "exec_shell_mode": "direct",
    "help_truncated": false,
    "help_truncated_at_size": null
  }
}
```

字段约束：

- `cache`：`"hit"` 不计配额、`"miss"` 计配额
- `prompt_version`：server 端 LLM prompt + 模型 + 解析逻辑的复合版本号；server 升级即整体冷启全局缓存。client 不发但收到后会 echo 在 `cpr --diag` 输出便于排错。
- `quota.reset_in_seconds`：到下次重置剩余秒数；client 文案优先用此字段显示"N 小时后重置"，避免本地时钟 / 时区库依赖。`reset_at` 仍保留作 ISO 锚。
- `result.candidates[].kind`：`id` | `flag` | `subcommand` | `enum`，client 渲染时分组
- `result.candidates[].desc`：≤80 字符；超出 server 自截
- `result.danger`：boolean；true 时 client 必走 y/N 确认（除非 `confirm_danger=never`）
- `result.exec_template`：危险命令的执行模板。**协议级安全约束**：
    - 仅允许 `{name}` 占位符，`name` 必须匹配 `^[a-z][a-z0-9_]*$`
    - 模板内非占位符字符必须是可打印 ASCII，**禁止 shell 元字符** ``[;&|<>$`\\]``，禁止换行；client 收到违反者 → 报 `INVALID_TEMPLATE` 退出码 5（不渲染、不执行）
    - 多个占位符按 `exec_template_args` 指定来源
- `result.exec_template_args`：占位符 → 来源映射；`from: "candidates"` 表示填充值必须是 `result.candidates[].token` 中 `kind` 匹配的某一个；`from: "user_input"` 表示 client 在 y/N 前先要用户输入（v1 仅允许 `from: "candidates"`，`user_input` 留 v2）
- `result.exec_shell_mode`：`"direct"`（默认，client 走 `Executor` 的 `direct` 模式 + `shlex.split`，不进 shell）| `"bash-lc-source"`（必须显式声明；client 走 `bash -lc 'source <init> && <cmd>'`）。**默认 direct，server 不显式标即 direct。**
- `help_truncated` / `help_truncated_at_size`：见请求字段；server 仅 echo

错误（4xx/5xx）：

```json
{
  "schema_version": "1",
  "error": {
    "code": "QUOTA_EXCEEDED",
    "message": "今日配额已用完",
    "retry_after_seconds": 28800
  }
}
```

错误码（**client 与 server 都用同一份**，client 据此决定退出码）：

| code | HTTP | 来源 | client 退出码 | 文案要点 |
| --- | --- | --- | --- | --- |
| `HELP_NOT_FOUND` | - | client-only | 2 | "未识别的命令 / 工具未安装：`<tool>`" |
| `HELP_TIMEOUT` | - | client-only | 2 | "`<tool> --help` 超时（>2s）" |
| `BAD_REQUEST` | 400 | server | 5 | 协议层失败（field 缺失） |
| `INVALID_CLIENT` | 422 | server | 5 | client_id / version 非法 |
| `INVALID_TEMPLATE` | 422 | client（验 server 响应）/ server（自查） | 5 | "AI 返回的执行模板不合法（含禁用字符或非法占位符），请重试" |
| `SCHEMA_MISMATCH` | 426 | server | 5 | "client 版本与 server 协议不匹配，请升级（client v{x} ↔ server v{y}）" |
| `QUOTA_EXCEEDED` | 429 | server | 4 | "今日 X/Y，N 小时后重置；可配 ~/.cpr/config 自部署" |
| `LLM_PARSE_FAILED` | 502 | server | 5 | "AI 解析失败：<reason>" |
| `LLM_TIMEOUT` | 504 | server | 3 | "AI 解析超时" |
| `SERVER_ERROR` | 500 | server | 3 | "server 内部错误，请稍后重试" |
| `NETWORK_UNREACHABLE` | - | client-only | 3 | "server 不可达，请检查 ~/.cpr/config" |

注：client-only 错误（`HELP_NOT_FOUND` / `HELP_TIMEOUT` / `NETWORK_UNREACHABLE`）**不进 server**，由 client 在本地阶段直接退出。`INVALID_TEMPLATE` 双侧都可触发（server 在 LLM 输出后自查、client 在收到响应后再校验，互相兜底）。

## 3. 缓存

### 3.1 缓存 key

```
sub_path_norm = "/".join([s.strip().lower() for s in sub_path if s.strip()])
help_norm     = help_text.replace("\r\n", "\n").rstrip().encode("utf-8")

key = sha256(
    schema_version       + "\n" +
    prompt_version       + "\n" +
    tool                 + "\n" +
    sub_path_norm        + "\n" +
    locale               + "\n"
).update(help_norm).hexdigest()
```

要点：

- **`prompt_version` 入 key**：server 升级 LLM / prompt / 解析逻辑后，自动整体冷启全局缓存（旧 key 不再命中）。client 端本地缓存也会随之失效。
- **`client_version` 不入 key**：client patch 不影响"AI 解析输出"，避免每次 client 升级击穿缓存。
- **`sub_path` normalize**：strip + lower + 去空字符串，再用 `/` join。算法稳定可重现。
- **`help_text` 行尾归一**：`\r\n → \n`，`rstrip()` 去尾随空白，再 utf-8 编码。否则 Linux/macOS 跑同一工具会因换行差异 miss。
- **client 端**：算 key 时 `prompt_version` 取**本地缓存里上一次响应的值**；若本地无缓存则 `prompt_version = "unknown"`，必然 miss 一次拉到 server，server 写回真实 `prompt_version` 后下次复用。

### 3.2 client 本地缓存

- 位置：`~/.cpr/cache/cache.sqlite`
- 表结构：
  ```sql
  CREATE TABLE entries (
    key TEXT PRIMARY KEY,
    schema_version TEXT NOT NULL,
    prompt_version TEXT NOT NULL,
    response_json TEXT NOT NULL,
    created_at INTEGER NOT NULL,
    hit_count INTEGER NOT NULL DEFAULT 0
  );
  ```
- 启动时执行 `DELETE FROM entries WHERE schema_version != ?`（`?` = 当前 client 期待的协议版本），清旧版残留 —— 解决跨版本读到不兼容 entry 的问题。
- 容量：默认 1000 条，LRU（按 `created_at + hit_count` 综合）驱逐
- 失效：无 TTL（key 已含 help_text + prompt_version，二者任一变就 miss）
- 命中时 client **不发请求**；server quota 不动

### 3.3 server 全局缓存

- 同 key 算法
- 后端：MVP 用 sqlite/postgres 任选（T-003 内决定）
- 跨用户共享：A 用户问过 `git status --help`，B 用户再问直接命中（**不计 B 的配额**）
- 写入只在 LLM 解析成功 **且** server 自查 `INVALID_TEMPLATE` 通过后
- 失效：手动清表 / TTL 30 天（防错误 LLM 输出长期固化）/ `prompt_version` 升级即整体冷启

## 4. 脱敏

### 4.1 client 出站脱敏（`args` 字段）

实现于 `cpr/cli/redact.py`，输入 `(tool, args)`，输出 `args_redacted`。

**规则结构**：

```yaml
# 内置（随 client 发布）
global:
  always_redact_flags:
    # 任意 tool 都掩码
    - --password
    - --api-key
    - --secret
    - --token
  hash_whitelist:
    # 命中即不脱敏
    - regex: '^[0-9a-f]{40}$'         # git sha1
    - regex: '^sha256:[0-9a-f]{64}$'  # docker digest
    - regex: '^[0-9a-f]{7,12}$'       # short git sha
    - regex: '^v?\d+(\.\d+)+(-\w+)*$' # 语义化版本号 / sdkman id
  hash_blob_threshold:
    # 长 base64 / hex 才脱
    min_length: 40
    must_contain_non_hex: true        # 全是 [0-9a-f] 不算（避免击中 sha）
    or_pattern: '[A-Za-z0-9+/]{40,}={0,2}' # base64
  user_at_host:
    only_local_user: true             # 仅当 user == $USER / $LOGNAME 才替换为 <USER>

per_tool:
  mysql:
    redact_flags: ["-p"]              # mysql -p 是密码
  ssh:
    redact_flags: ["-i"]              # ssh -i <key> 路径
  scp:
    redact_flags: ["-i"]
  # docker / kubectl 不在此列：-p 是端口，不脱敏

# 用户私有（~/.cpr/config）
user:
  extra_global_flags: []
  extra_per_tool: {}
```

**flag 形式**：同时支持空格分隔（`--token abc`）与等号合并（`--token=abc`）。等号形式由 `redact.py` 在切分前预处理。

**脱敏值**：统一替换为 `<REDACTED>`；保留原 flag 名以便 `usage` 渲染对齐。

### 4.2 server 出站镜像脱敏

server 在 LLM 返回后、写缓存前，对响应中以下字段**应用同一份脱敏**（global + 命中 `tool` 的 per_tool）：

- `result.usage`
- `result.candidates[].token`（仅 `kind ∈ {id, enum}` 时不脱；`kind ∈ {flag}` 跳过；其他按字符串处理）
- `result.exec_template`

防止 LLM 把 help_text 里的示例 `--password mypass` 复述出来。

### 4.3 脱敏配置文件

`~/.cpr/config` 中 `redact` 段对应上述 `user` 区域；client 启动时合并 `global + per_tool + user`。配置示例：

```yaml
redact:
  extra_global_flags: ["--my-private-token"]
  extra_per_tool:
    docker:
      redact_flags: ["--registry-password"]
  keep_user: false
```

### 4.4 测试义务

- 每条 `global.always_redact_flags`、每条 `per_tool` 至少 2 例（命中 / 不命中）。
- `--key=value` 与 `--key value` 两种形式必测。
- `hash_whitelist` 全部 regex 至少 1 正例。
- `user_at_host` 三例：`$USER@host` / 别人@host / `git@github.com`。
- 长 base64 / hex 边界：39 字符不脱、40 字符脱、纯 hex 不脱、混合脱。

## 5. 配额（server 端）

- 表 `quota_usage(client_id TEXT, day TEXT, count INTEGER)` 主键 `(client_id, day)`
- `day` = UTC 当日（避免时区抖动；client 文案上做转换，并优先用 `quota.reset_in_seconds` 显示）
- LLM 调用前 `count >= limit` → 返回 `QUOTA_EXCEEDED`，**不调 LLM**
- LLM 成功后 `count += 1`
- 自部署侧：`limit` 配置为 `-1` 即不限

## 6. 危险命令处理

- "危险"由 server 在响应中标 `danger: true` + `danger_reason` + `exec_template` + `exec_template_args` + `exec_shell_mode`
- client 收到响应后**先做模板自查**（§2.1 `result.exec_template` 字段约束）；不通过 → `INVALID_TEMPLATE` 退出码 5，不渲染、不执行
- 通过自查 → 渲染 [1]–[5]，追加：`确认执行？(y/N)`，默认 N
- y → client 按 `exec_shell_mode` 执行：
    - `direct`：`shlex.split(rendered_cmd)` + `subprocess.run`，**不进 shell**
    - `bash-lc-source`：`Executor` 的 `bash-lc-source` 模式（沿用 T-001）
- N / 直接回车 → 仅展示，进程退出码 0
- **危险确认配置**（client 侧 `~/.cpr/config`）：
    ```yaml
    client:
      confirm_danger: always   # always | once | never
    ```
    - `always`（默认）：每次危险命令都 y/N
    - `once`：单次会话内对同一 `(tool, sub_path)` 二次执行不再确认（v1 实现：客户端进程内 set；不持久化跨进程）
    - `never`：相当于 `--yes`，跳过 y/N 直接执行
- CLI 短路：`--yes` / `-y` 等同于 `confirm_danger=never`，仅本次调用生效

## 7. 接口演进

- 协议字段加新键：向前兼容（client 忽略未知键、server 忽略未知字段）
- 字段类型变更 / 删除：必须先在 ≥1 发布周期把字段标记为 `deprecated`（响应中保留 + 文档标记），下个版本才能删；删除时升 `schema_version`
- `schema_version` 升版后，server 必须同时支持新旧两版至少一个发布周期
- 错误码扩充：新增 code → 旧 client 退出码兜底为 5
- 默认 timeout：help 抓取 2s、server 请求 5s；二者均可由 `~/.cpr/config` 覆盖（`server.timeout_seconds` / `client.help_timeout_seconds`）
