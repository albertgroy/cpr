## 契约缺口摘要（详见原始评审存档）

### CommandNode

- `id` 命名规则：`^[a-z][a-z0-9]*(\.[a-z0-9_{}-]+)*$`，参数段固定 `{name}`。
- 不变量 `id == tokens.join(".")`，`{name}` 在 id 中保留花括号。
- `children` 为 ID 引用；加载分两阶段 parse → link。
- 默认 `execute.command = tokens.join(" ")`，可显式覆盖；`{name}` 占位符必须来自路径上的 `param` 节点。
- `param.source` 取值枚举（首批两个）：`sdk.candidates.java.identifiers`、`sdk.installed.java.versions`；新增 source 必须更新枚举表。
- 允许中间节点同时携带 `execute`（如 `sdk current`）；UI 同时显示候选与"Enter 执行"。

### CommandSession

- 解析使用最长前缀匹配，未匹配尾部进入 `unparsedTail`，`currentNode` 指最长匹配节点。
- `lastResult` 至少保留最近 1 条；建议环形缓冲 N=5；保证 `/fix` 可用。
- Backspace 仅在输入区为空时弹 token；非空走普通字符删除。
- Esc：输入区非空 → 清空；输入区空 → 弹 token。

### CandidateProvider

- 接口异步：`async def provide(...) -> Candidates`；UI 占位 `(loading…)` + `app.invalidate()`。
- 动态候选 ttl=60s；`sdk install/use/default` 执行成功后强制失效 `sdk.installed.java.versions` 与 `sdk.candidates.java.identifiers`；不变量"修改环境的 Executor 调用必须发出失效事件"。
- 解析失败返回空列表 + `error: { code, message }`，UI 显示"动态候选解析失败：<原因>"。
- `label` / `desc` 走 i18n + fallback。

### Executor

- 超时：`direct` 默认 30s；`bash-lc-source` 默认 120s；超时 `ok=false, exitCode=-1, errorKind="timeout"`。
- 编码：`LC_ALL=C.UTF-8`（Linux）/ `en_US.UTF-8`（macOS fallback）；utf-8 with errors="replace"。
- 默认 pipe 模式；TTY 模式留作后续 issue。
- `cwd=$HOME`；env 白名单 `HOME / PATH / LANG / LC_ALL / SDKMAN_DIR`。
- stderr 与 stdout 分流保留，UI 渲染时按顺序合并并染色。
- `bash-lc-source` 路径优先 `$SDKMAN_DIR/bin/sdkman-init.sh`，缺省回退 `~/.sdkman`。
- 取消语义：`cancel()` → SIGINT → 2s SIGTERM → 2s SIGKILL。

### SlashCommand

- Parser 文法：`^/(\w+)(?:\s+(.*))?$`，name 小写，args 取整行。
- `/back` 与 Backspace 走同一 `session.popToken()`。
- `/clear` 仅清显示缓冲，保留 `lastCommand` / `lastResult`。
- 未知 slash 不进入历史，仅显示提示。

### 跨契约

- 加载失败 → 终端打印诊断 + 退出码 2，禁止半启动。
- locale 解析顺序：`CPR_LOCALE` > 用户配置 > `LANG` > `en-US`。
- 日志路径 `~/.cpr/logs/cpr.log`，warn 起记，启动时显示路径。
