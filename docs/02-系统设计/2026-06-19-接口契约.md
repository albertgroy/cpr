# 接口契约（2026-06-19）

## 范围

定义 CPR 原型阶段（阶段 1–3）的核心模型对外契约。本文档与 `docs/01-需求分析/产品功能详细描述.md`、`docs/03-项目规划/任务规划.md` 互相引用。

## CommandNode

### 输入

- `id`：层级唯一字符串，例 `sdk.list.java`。
- `tokens`：从根到当前层的 token 数组，例 `["sdk", "list", "java"]`。
- `title`：i18n 字典 `{ "zh-CN": ..., "en-US": ... }`。
- `desc`：i18n 字典。
- `children`：子节点 ID 数组。
- `param`：可选，动态参数节点的占位符与 `source`（候选源 ID）。
- `execute`：可选，叶子节点执行命令字符串或 shell 调用模板。

### 不变量

- 同一层级中 token 唯一。
- `param` 节点的 `tokens` 末尾必须形如 `"{name}"`。
- `execute` 仅出现在可执行节点（叶子或中间执行点）。

### 错误

- 不存在的子节点 ID → 加载阶段抛错。
- 缺失语言 → fallback 到 `en-US` 或 `zh-CN`。

## CommandSession

### 状态

- `currentTokens`：当前已构造 token 数组。
- `currentNode`：当前 token 路径对应的 CommandNode。
- `lastCommand`：上一次执行的完整命令。
- `lastResult`：`{ ok, stdout, stderr, exitCode }`。

### 状态迁移

- 输入 token：追加到 `currentTokens`，重新解析 `currentNode`。
- Backspace（输入区为空）：弹出末尾 token。
- Enter：若 `currentNode` 可执行 → 执行；否则若有候选选中 → 追加 token。
- Esc：清空当前未提交输入或回退一层。

### 错误

- 解析失败（无对应节点）→ 标记为"未识别命令"，仍允许直接执行真实 shell。

## CandidateProvider

### 输入

- `nodeId`：当前 CommandNode ID。
- `session`：CommandSession（用于动态候选所需的上下文）。

### 输出

- `candidates: [{ token, label, desc, source }]`，其中 `source` 为 `static` 或 `dynamic`。

### 动态候选示例

- `sdk list java identifiers`：解析 `sdk list java` 输出。
- 已安装版本：解析 `~/.sdkman/candidates/java/` 目录。

## Executor

### 输入

- `command`：完整命令字符串。
- `shellMode`：`direct` 或 `bash-lc-source`（用于 SDKMAN shell function）。

### 输出

- `{ ok, stdout, stderr, exitCode, durationMs }`。

### 不变量

- 不静默吞错；失败时 `ok=false` 且保留 stderr。
- shell function 命令必须经过 `bash -lc 'source ~/.sdkman/bin/sdkman-init.sh && <command>'`。

## Slash Command

### 内置集合

`/ai` / `/explain` / `/fix` / `/help` / `/clear` / `/back`

### 输入

- 主输入区文本以 `/` 起始。
- Parser 抽出 `name` 与 `args`。

### 输出

- 修改 Session（如 `/back` 弹 token、`/clear` 清空内容区）。
- 或调用 AI provider 并把结果写入内容区。

### 错误

- 未知 slash → 显示提示并保留输入。
