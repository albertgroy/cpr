## P3-T001-1 · `make_composite_resolver` 多 param 子节点选择

- 现状：在 `cpr/tui/app.py:make_composite_resolver` 中，"current.children 中找首个 param 子节点"用 `for child in node.children: if child.param is not None: break` —— 多 param 子节点时只命中第一个。
- 影响：当前 SDKMAN 命令树（`sdk install java`、`sdk use java`、`sdk default java`）每个父节点最多 1 个 param 子，无触发。
- 触发条件：T-002+ 若扩展多 vendor 参数（例如 `sdk install <java|kotlin> <id>`）会暴露问题。
- 修复方向：按当前 token 上下文（用户已经追加了哪些 token）区分；或在 CommandNode 里给 param 子节点声明优先级。
- 优先级：P3，T-002 命令树扩展时一并设计。

## P3-T001-2 · stderr 在 content_buffer 中无着色

- 现状：`cpr/tui/app.py:run_app.execute` 把 `result.stderr` 直接拼接进 `content_buffer`，未应用 `STYLE` 中已声明的 `content.stderr` / `content.error` 类。
- 契约：模型层要求"不静默吞错"已满足，UI 着色属于"Loop 2 体感优化"。
- 修复方向：把 content_buffer 改为 FormattedTextControl，或者按段落 tag。
- 优先级：P3，T-003 体感优化时一并做。

## P3-T001-3 · `dispatch_slash` 尾换行兜底

- 现状：`cpr/tui/app.py:dispatch_slash` 拼 `f"{text}\n{result.message}\n"`；`SlashCommandResult.message` 当前都不带尾换行，但若未来某条 slash 返回多行带尾 `\n`，内容区会出现空行。
- 修复方向：`result.message.rstrip("\n")` 兜底，或在 `SlashCommandResult` 文档化"message 不得以换行结束"约束。
- 优先级：P3，下次接 slash 命令（如 `/ai` 真 LLM 实装）时一并处理。

## P3-T001-4 · `tests/tui/__init__.py` 空文件

- 现状：终验补丁创建的 `tests/tui/__init__.py` 是空文件；CI 行为正常。
- 影响：走查时不易判断"是否还有未提交的 fixture / 共用 helper"。
- 修复方向：要么显式声明 `# Test package` 一行注释，要么把可能复用的 `_FakeBuffer`、`_setup` 提到 `tests/tui/conftest.py` 或 `tests/tui/__init__.py` 中作为 fixture。
- 优先级：P3，下次 tui 测试规模上来后整治。
